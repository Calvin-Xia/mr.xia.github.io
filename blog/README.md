# 博客 JSON 文件使用指南

本目录包含博客系统所需的 JSON 配置文件及其示例。

## 文件说明

### 1. blog-files.json
**用途**：存储所有博客文章的 HTML 文件路径列表

**格式**：字符串数组
```json
[
  "blog/20251231-2025年度总结.html",
  "blog/20251215-前端开发最佳实践.html"
]
```

**命名规范**：
- 建议使用日期前缀：`YYYYMMDD-文章标题.html`
- 例如：`20251231-2025年度总结.html`
- 日期格式：`YYYY`（年）`MM`（月）`DD`（日）

### 2. blog-metadata.json
**用途**：存储所有博客文章的元数据信息，用于搜索、筛选和展示

**格式**：对象数组
```json
[
  {
    "id": "20251231-2025年度总结",
    "title": "2025年度总结",
    "excerpt": "文章摘要...",
    "category": "生活总结",
    "tags": ["2025", "年度总结", "大学"],
    "date": "2025-12-31",
    "filePath": "blog/20251231-2025年度总结.html"
  }
]
```

**字段说明**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 文章唯一标识符，建议使用日期+标题 |
| title | string | 是 | 文章标题 |
| excerpt | string | 是 | 文章摘要，建议 100-200 字 |
| category | string | 是 | 文章分类 |
| tags | array | 是 | 文章标签数组，建议 3-5 个 |
| date | string | 是 | 发布日期，格式：YYYY-MM-DD |
| filePath | string | 是 | 文件路径，从网站根目录开始 |
| author | string | 否 | 作者名称 |
| readTime | string | 否 | 预计阅读时间，格式：X分钟 |
| featured | boolean | 否 | 是否为精选文章 |

## 添加新文章步骤

### 步骤 1：创建 HTML 文件
在 `blog/` 文件夹中创建新的 HTML 文件，例如：`blog/20250101-新文章.html`

### 步骤 2：添加文件路径
在 `blog-files.json` 中添加文件路径：
```json
[
  "blog/20251231-2025年度总结.html",
  "blog/20250101-新文章.html"
]
```

### 步骤 3：添加元数据
在 `blog-metadata.json` 中添加对应的元数据：
```json
[
  {
    "id": "20250101-新文章",
    "title": "新文章标题",
    "excerpt": "文章摘要...",
    "category": "分类名称",
    "tags": ["标签1", "标签2"],
    "date": "2025-01-01",
    "filePath": "blog/20250101-新文章.html"
  }
]
```

### 步骤 4：测试
在浏览器中打开 `statement.html`，检查文章是否正常显示。

## 注意事项

1. **唯一性**：确保 `id` 字段唯一，不能重复
2. **一致性**：`filePath` 必须与实际文件路径完全一致
3. **日期格式**：`date` 格式必须为 `YYYY-MM-DD`
4. **数组格式**：`tags` 必须是数组格式
5. **摘要长度**：`excerpt` 建议控制在 200 字以内
6. **同步更新**：添加新文章后，记得同步更新 `blog-files.json`

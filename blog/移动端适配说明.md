# HTML文件移动端适配与图片懒加载说明

## 自动转换工具
运行 `convert.py` 可自动转换当前目录下所有HTML文件：
```bash
python convert.py
```

也可指定文件：
```bash
python convert.py 文件1.html 文件2.html
```

### 工具功能
1. **移动端适配**：添加viewport标签、修改容器宽度
2. **图片懒加载**：自动为img标签添加懒加载支持
3. **Favicon图标**：自动添加网站favicon标签
4. **引用块修复**：自动修复手机端蓝色引用块过窄的问题

---

## 图片懒加载功能

### 工作原理
工具会自动将图片的 `src` 属性转换为 `data-src`，并注入 Intersection Observer 脚本实现懒加载。

### 转换效果
**转换前：**
```html
<img src="image.jpg" alt="图片描述">
```

**转换后：**
```html
<img loading="lazy" class="lazy-img" data-src="image.jpg" alt="图片描述">
```

### 懒加载优势
- 提升页面加载速度
- 节省带宽流量
- 改善用户体验

## 手动修改方法

## 问题
dooc社区导出的HTML文件未动态适配手机端浏览器。

## 原因
1. 缺少viewport meta标签
2. 内容容器设置了固定宽度 `width: 750px`

## 解决方案

### 步骤一：添加viewport标签
在每个HTML文件的 `<head>` 标签内添加viewport声明。

**修改位置：** 找到 `<meta charset="utf-8">` 这一行，在其后添加一行。

**修改前：**
```html
<head>
  <meta charset="utf-8">
  <title>页面标题</title>
```

**修改后：**
```html
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>页面标题</title>
```

### 步骤二：添加favicon标签
在 `<head>` 标签内添加网站图标链接。

**修改位置：** 在 `<meta charset="utf-8">` 之后添加。

**修改后：**
```html
<head>
  <meta charset="utf-8">
  <link rel="icon" type="image/png" href="../storage/icon.png">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>页面标题</title>
```

> 注意：blog目录下的文件需使用 `../storage/icon.png` 路径，根目录文件使用 `storage/icon.png`。

### 步骤三：修改容器宽度（关键）
找到 `<body>` 标签后的第一个 `<div>`，修改样式。

**修改前：**
```html
<div style="width: 750px; margin: auto; padding: 20px;">
```

**修改后：**
```html
<div style="max-width: 750px; width: 100%; margin: auto; padding: 20px; box-sizing: border-box;">
```

> 注意：必须添加 `box-sizing: border-box`，否则 padding 会导致实际宽度超出屏幕。

## 说明
| 属性 | 作用 |
|------|------|
| `width=device-width` | 页面宽度等于设备屏幕宽度 |
| `initial-scale=1.0` | 初始缩放比例为1（不缩放） |
| `max-width: 750px` | 大屏幕上最大宽度限制为750px |
| `width: 100%` | 小屏幕上自动适应屏幕宽度 |
| `margin: auto` | 水平居中显示 |
| `padding: 20px` | 内边距20px |
| `box-sizing: border-box` | 包含padding在内计算宽度 |

## 引用块蓝框宽度异常说明

### 现象
部分 blog 页面在手机端会出现蓝色引用块明显偏窄的情况，例如：

```text
时间：20260203 下午 17:00~18:00
地点：重庆市万州第二高级中学高中部中心广场
```

这类内容在桌面端不明显，但在手机端会看起来像“蓝框宽度异常”。

### 根因
问题不在于页面整体横向溢出，而在于 `blockquote` 保留了浏览器默认的左右外边距。

- 浏览器默认会给 `blockquote` 添加左右 `40px` margin
- 在手机视口下，这会显著压缩引用块可用宽度
- 当引用块带有蓝色左边框时，视觉上就会像只有一小块蓝框

### 自动修复策略
`convert.py` 会自动在 `</head>` 前注入一段带唯一标记的移动端样式，仅在窄屏下生效：

- 为 `blockquote` 和 `.md-blockquote` 去掉左右默认 margin
- 增加 `box-sizing: border-box`
- 保证引用块宽度按正文列宽计算
- 为 `blockquote > p` 和 `.md-blockquote-p` 添加安全换行规则，避免英文串、链接或时间地点行挤压布局

该注入是幂等的，多次运行 `python convert.py` 不会重复插入相同样式。

### 验证口径
在手机视口下检查 blog 页面时，应满足：

- 蓝色引用块宽度与正文列宽一致
- 不再保留浏览器默认的左右 `40px` 空白
- 页面无新增横向滚动

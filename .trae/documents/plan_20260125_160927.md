## 实现方案1：使用 JSON 文件管理文章列表

### 主要修改内容：

1. **创建 blog-files.json 文件**

   * 在 blog 文件夹下创建 `blog-files.json` 文件

   * 使用 JSON 数组格式存储文章路径列表

   * 包含现有的文章路径：`["blog/20251231-2025年度总结.html"]`

2. **修改 JavaScript 代码**

   * 移除硬编码的 `blogFiles` 数组

   * 添加 `loadBlogFilesList()` 方法从 JSON 文件读取文章列表

   * 修改 `loadAllArticles()` 方法，先读取 JSON 文件再加载文章

   * 添加错误处理：JSON 文件加载失败时的降级方案

3. **错误处理和容错**

   * JSON 文件加载失败时显示友好错误提示

   * JSON 格式错误时在控制台输出警告

   * 文章加载失败时跳过该文章，继续加载其他文章

4. **文档说明**

   * 在 JSON 文件中添加注释说明如何添加新文章

   * 提供示例格式

### 实现细节：

* JSON 文件路径：`blog/blog-files.json`

* JSON 格式：字符串数组，每个元素是文章的相对路径

* 读取方式：使用 `fetch()` API

* 错误降级：如果 JSON 加载失败，显示错误提示而不是空白页面
